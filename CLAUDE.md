# Prisma for Rust - Project Guide

## Project Overview

**Prisma ORM for Rust** - A type-safe database client generator that reuses Prisma's query execution engine while providing Rust-native macros for zero-boilerplate code generation.

**Architecture**: Pure macro-based (no CLI code generation). Schema file → Rust types → Type-safe builders → Database queries.

**Status**: Active development. Working MVP with Phase 2 improvements in progress.

---

## How It Works: Two-Level Macro Architecture

### Level 1: `init!("schema.prisma")` - Schema to Rust Types
Compile-time function-like macro that:
1. Reads your `schema.prisma` file
2. Parses with official PSL parser (`psl::validate()`)
3. Walks models/enums using PSL's walker API (`db.walk_models()`, `db.walk_enums()`)
4. Generates Rust structs and enums with `#[prisma_model]` attribute pre-applied
5. Extracts datasource URL and generates `db::client()` function
6. Wraps everything in `pub mod db { ... }`

**Usage:**
```rust
use prisma_macros::init;

init!("schema.prisma");

#[tokio::main]
async fn main() -> Result<()> {
    let client = db::client().await?;  // Auto-initialized from datasource
    // ... use db::User, db::Post, db::Role, etc.
}
```

### Level 2: `#[prisma_model]` - Builders and Query Factories
Attribute macro that operates on structs generated by `init!()`:
1. Extracts field metadata via `#[prisma(...)]` attributes
2. Generates type-safe builders:
   - `{Model}WhereBuilder` - filtering with AND/OR/NOT
   - `{Model}UniqueWhereBuilder` - unique field filtering
   - `{Model}SelectBuilder` - field selection
   - `{Model}IncludeBuilder` - relation inclusion
   - `{Model}DataBuilder` - create/update data
3. Generates thin wrappers:
   - `{Model}ReadBuilder` - find_many, find_unique, find_first, etc.
   - `{Model}UniqueReadBuilder` - find_unique with type-safe where
   - `{Model}WriteBuilder` - create, update, delete, upsert
   - `{Model}CountBuilder`, `{Model}AggregateBuilder`, `{Model}GroupByBuilder`
4. Generates query factory:
   - `{model}()` function returning `{Model}Query`
   - All 14 CRUD operations

## Current Working Features

### ✅ What Works
- All CRUD operations (find_many, find_unique, find_first, create, update, delete, create_many, update_many, delete_many, upsert)
- Aggregations (count, aggregate, group_by)
- Type-safe filters per field type (StringFilter, IntFilter, BoolFilter, EnumFilter, FloatFilter, DateTimeFilter)
- Compile-time operator validation (e.g., `.contains()` only on strings, `.gt()` only on numbers)
- Enums with Display + PrismaValue conversion
- Model structs with serde serialization
- Auto-initialized PrismaClient from schema datasource
- Zero boilerplate - just `init!()` and use generated code

## Project Structure

### Key Directories

**prisma_core/src/**
- `builder.rs` - Base traits: `Executable`, `Filterable`, `Selectable`, `FilterBuilder`
- `read.rs` - `ReadBuilder` for all find operations
- `write.rs` - `WriteBuilder` for create/update/delete/upsert
- `aggregate.rs` - `CountBuilder`, `AggregateBuilder`, `GroupByBuilder`
- `filters.rs`, `filter_builders.rs` - Type-safe filter system
- `prelude.rs` - Re-exports all traits for scope access in generated code
- `client.rs` - `PrismaClient` initialization

**prisma-macros/src/**
- `lib.rs` - `init!()` macro (schema → Rust types) and `#[prisma_model]` (types → builders)
- `model_gen.rs` - Generates structs/enums from PSL using walker API
- `builder_gen.rs` - Generates where/select/include/data builders
- `wrapper_gen.rs` - Generates thin wrapper builders (read, write, count, aggregate, group_by)
- `query_gen.rs` - Generates query factories and operations
- `model_analysis.rs` - Field metadata extraction from structs
- `codegen_orchestrator.rs` - Coordinates model_gen output with builder generation
- `utils.rs` - Helper functions (type detection, filter mapping)

**example-app/**
- `schema.prisma` - Prisma schema (source of truth)
- `src/main.rs` - Usage example with `init!()` macro
- `Cargo.toml` - Dependencies

## Development Workflow

### Adding a New Feature

1. **Schema-level**: Add to `example-app/schema.prisma`
   ```prisma
   model Todo {
     id    String @id
     title String
     done  Boolean @default(false)
   }
   ```

2. **Run**: `cargo run -p example-app` - `init!()` auto-generates all code

3. **Use in code**: `db::todo().create("Buy milk").exec(&client).await?`

### Debugging

- **Type-safe filters not working?** Check `prisma_core/src/filters.rs` for trait definitions
- **Builder generation issues?** Check `prisma-macros/src/builder_gen.rs`
- **Schema parsing failing?** Check `model_gen.rs` walker usage or `codegen_orchestrator.rs`
- **Runtime execution errors?** Check `prisma_core/src/read.rs` or `write.rs`

## Important Architecture Decisions

1. **Two-level macros** (not single): Allows schema parsing complexity to be separate from builder generation complexity
2. **Official PSL parser**: Uses `psl::validate()` and walker API to extract models/enums accurately
3. **Compile-time type safety**: Filter operators validated at compile-time via trait implementation
4. **Thin wrappers**: `ReadBuilder`, `WriteBuilder` etc. are minimal - they delegate to generic builders
5. **No CLI code generation**: Everything happens at compile-time via macros
6. **Automatic client initialization**: `db::client()` function auto-created from schema datasource

## Key Principles

- ✅ **Single source of truth**: Schema file drives everything
- ✅ **Zero boilerplate**: Just `init!()` and write queries
- ✅ **Compile-time safety**: Wrong code → compile error, not runtime error
- ✅ **Clean DX**: Fluent API, type inference, helpful error messages
- ✅ **Practical features first**: 80/20 approach - focus on what matters most
